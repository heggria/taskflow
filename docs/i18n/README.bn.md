**সাবএজেন্ট (subagents) হলো ফায়ার-অ্যান্ড-ফরগেট। টাস্কফ্লো (Taskflows) ফায়ার করে, ফ্যান আউট করে, পজ করে, গেট করে, রিজিউম করে এবং নিজেদের একটি কমান্ড হিসেবে সেভ করে।**

আপনি ইতিমধ্যেই বিল্ট-ইন সাবএজেন্ট টুলের `task` / `tasks` / `chain` জানেন। `pi-taskflow` **একই** শর্টহ্যান্ডে কথা বলে — ফলে আপনার existing ডেলিগেশনগুলো তাৎক্ষণিকভাবে **ট্র্যাকযোগ্য, পুনরায় শুরুযোগ্য এবং একটি এক-শব্দের `/tf:<name>` কমান্ড হিসেবে সেভযোগ্য** হয়ে ওঠে। যখন শর্টহ্যান্ড আর যথেষ্ট না হয়, তখন পূর্ণ DSL আপনাকে একটি বাস্তব DAG দেয়: ডজন ডজন আইটেমের উপর ডায়নামিক ফ্যান-আউট, কন্ডিশনাল রাউটিং, কোয়ালিটি গেট, হিউম্যান অ্যাপ্রুভাল, রিট্রাই এবং একটি কঠোর স্পেন্ড সিলিং।

এবং পুরো সময় জুড়ে, **শুধুমাত্র চূড়ান্ত ফেজ আপনার কনভারসেশনে পৌঁছায়।** প্রতিটি মধ্যবর্তী ট্রান্সক্রিপ্ট রানটাইমে থেকে যায়, কখনো আপনার কনটেক্সট উইন্ডোতে নয়।

## কেন এটি বিদ্যমান

র সাবএজেন্টের সাথে আপনি যে দেয়ালে ঠোক্কর খান তা হল: আপনি একটি মাল্টি-স্টেপ প্ল্যান প্রোজে বর্ণনা করেন, মডেল প্রতিবার সেটি পুনরায় ডিরাইভ করে, মধ্যবর্তী ট্রান্সক্রিপ্ট আপনার কনটেক্সট প্লাবিত করে, এবং যেই একটি মডেল কল ব্যর্থ হয় আপনি শূন্য থেকে শুরু করেন। কোনো পুনর্ব্যবহার নেই, কোনো পুনরুদ্ধার নেই, কোনো কাঠামো নেই।

`pi-taskflow` প্ল্যানটিকে **প্রম্পট থেকে বের করে একটি ডিক্লারেটিভ ডেফিনিশনে নিয়ে আসে।** রানটাইম DAG, লুপ, রিট্রাই এবং মধ্যবর্তী স্টেটের মালিক হয়। আপনি একবার একটি পাইপলাইন ডিক্লেয়ার করেন এবং এটি শতবার চালান — নাম ধরে।

> যখন একটি কাজের জন্য শাখা-প্রশাখা ও রিভিউ গেটসহ বারোটি ধাপ প্রয়োজন, তখন আপনার দরকার অর্কেস্ট্রেশন — ভাগ্যবান প্রম্পটিং নয়।

| | subagent (বিল্ট-ইন) | **pi-taskflow** |
|---|---|---|
| **কে চালায়** | মডেল, প্রতি পালায় | রানটাইম, একটি ডেফিনিশন থেকে |
| **টপোলজি** | চেইন / সমান্তরাল | **স্তরযুক্ত কনকারেন্সি + রাউটিং সহ DAG** |
| **মধ্যবর্তী ফলাফল** | আপনার কনটেক্সট উইন্ডোতে | **রানটাইমে — আপনার কনটেক্সটে নয়** |
| **স্কেল** | হাতে গোনা কয়েকটি টাস্ক | **ডজন ডজন আইটেমের উপর ডায়নামিক `map` ফ্যান-আউট** |
| **পুনর্ব্যবহারযোগ্য** | প্রতিবার পুনরায় বর্ণিত | **`/tf:<name>` হিসেবে সেভ করা** |
| **পুনরায় শুরুযোগ্য** | ✗ | **✓ ক্রস-সেশন — ক্যাশেড ফেজ স্বয়ংক্রিয়ভাবে স্কিপ** |
| **কোয়ালিটি গেট** | ✗ | **`gate` ফেজ যা `VERDICT: BLOCK` এ থামে** |
| **কন্ডিশনাল রাউটিং** | ✗ | **`when` গার্ড + `join: any` OR-জয়েন** |
| **ফল্ট টলারেন্স** | ✗ | **প্রতি-ফেজ `retry` + ট্রানজিয়েন্ট ত্রুটিতে অটো-রিট্রাই** |
| **হিউম্যান-ইন-দ্য-লুপ** | ✗ | **`approval` ফেজ (অ্যাপ্রুভ / রিজেক্ট / এডিট)** |
| **খরচ নিয়ন্ত্রণ** | ✗ | **রান-ওয়াইড `budget` (USD / টোকেন ক্যাপ)** |
| **কম্পোজিশন** | ✗ | **`flow` ফেজ সেভ করা সাব-ফ্লো চালায়** |
| **লাইভ প্রোগ্রেস** | চলাকালীন অস্বচ্ছ | **সময় + খরচ সহ লাইভ DAG রেন্ডার** |
| **এরগোনমিক্স** | প্রতিবার ইনলাইন JSON | **শর্টহ্যান্ড (`task`/`tasks`/`chain`) *বা* DSL** |

এটি সাবএজেন্ট টুলকে প্রতিস্থাপন করে না। এটি আপনার সাবএজেন্টদের একটি DAG, একটি মেমোরি এবং একটি নাম দেয়।

## অন্যান্য Pi এক্সটেনশনের সাথে তুলনা

Pi ইকোসিস্টেমে এখন **20+ ডেলিগেশন, ওয়ার্কফ্লো এবং অর্কেস্ট্রেশন এক্সটেনশন** রয়েছে — প্রতিটি নিজস্ব কাজে দুর্দান্ত। এখানে `pi-taskflow` কোথায় অবস্থান করে তার একটি সৎ মানচিত্র দেওয়া হলো (প্রতিটি প্যাকেজের সর্বশেষ npm রিলিজ, জুন 2026 অনুযায়ী যাচাইকৃত)। সম্পূর্ণ বিশদ — প্রতিটি প্যাকেজের শক্তি *এবং* দুর্বলতা — দেখতে: [`PI-ECOSYSTEM.md`](./PI-ECOSYSTEM.md)। বিস্তৃত, নন-Pi ল্যান্ডস্কেপ (LangGraph, Temporal, CrewAI, Mastra…) দেখতে: [`COMPETITORS.md`](./COMPETITORS.md)।

**কিভাবে নির্বাচন করবেন:**

- **`@pi-agents/orchid`** ইকোসিস্টেমের সবচেয়ে ফিচার-সম্পূর্ণ অর্কেস্ট্রেটর (DAG + ওয়ার্কট্রি + Ralph লুপ + এজেন্ট মেলবক্স) — তবে এর DSL একটি *ফিক্সড* ৯-ফেজ পাইপলাইন, এতে রানটাইম ডিপ + jiti আছে এবং এটি বিটা। `pi-taskflow` বেছে নিন যখন আপনি **নিজের গ্রাফ সংজ্ঞায়িত করতে চান** (একটি ওপিনিয়নেটেড গ্রাফ গ্রহণ না করে), **শূন্য ডিপেন্ডেন্সি** এবং একটি এক-কমান্ড ইনস্টল সহ।
- **`pi-crew` / `ultimate-pi`** ভারী যায় — ওয়ার্কট্রি আইসোলেশন, টেকসই অ্যাসিঙ্ক টিম, মাল্টি-টায়ার গভর্নেন্স। যদি হালকা, ডিক্লারেটিভ এবং জিরো-ডিপেন্ডেন্সি চান, এটি সেই প্রজেক্ট।
- **`@zhushanwen/pi-workflow`** আত্মায় সবচেয়ে কাছের এবং এটিও জিরো-ডিপ, তবে আপনি ওয়ার্কফ্লো **JavaScript স্ক্রিপ্ট** হিসেবে লেখেন। `pi-taskflow`-এর **ডিক্লারেটিভ JSON DSL** বেশি নিরাপদ এবং অডিটযোগ্য, এবং এর **ফেজ-লেভেল ইনপুট-হ্যাশ রিজিউম** কল-ক্যাশ ডিডাপের চেয়ে বেশি গ্রানুলার।
- **`@fiale-plus/pi-rogue-orchestration`**-এ একটি বাস্তব **লুপ-অ্যান্টিল-ডান** রয়েছে (একটি ফিচার যা `pi-taskflow`-এ এখনও নেই)। যদি আপনার কাজ "লক্ষ্য পূরণ না হওয়া পর্যন্ত চলতে থাকে" হয়, এটি দেখার মতো; `pi-taskflow` পরিবর্তে *গঠিত, শাখাপ্রশাখাময়* পাইপলাইনের জন্য।
- **`pi-subagents` / `@gotgenes/pi-subagents`** হল অ্যাড-হক "এই ডিফে reviewer ব্যবহার করুন" ডেলিগেশন এবং ব্যাকগ্রাউন্ড কাজের জন্য পরিণত পছন্দ। `pi-taskflow` তখনই যখন সেই ডেলিগেশনগুলোকে একটি *পুনরাবৃত্তিযোগ্য, পুনরায় শুরুযোগ্য পাইপলাইন* হতে হবে।
- **`pi-pipeline` / `pi-agent-flow`** ওপিনিয়নেটেড, ফিক্সড ফ্লো সরবরাহ করে। `pi-taskflow` একটি *খালি ক্যানভাস* সরবরাহ করে: আপনি (বা মডেল) সেই গ্রাফ ডিক্লেয়ার করেন যা কাজের জন্য উপযুক্ত।

> সৎ এক-লাইনারের উত্তর: **`pi-taskflow` হল একমাত্র Pi এক্সটেনশন যা আপনাকে একটি ডিক্লারেটিভ, রিজিউমেবল, DAG-আকৃতির সাবএজেন্ট পাইপলাইন দেয় যা আপনি একটি এক-শব্দের কমান্ড হিসেবে সেভ করেন — শূন্য রানটাইম ডিপেন্ডেন্সি এবং কনটেক্সট আইসোলেশন সহ।** পরিচিত ফাঁক যা পরবর্তীতে বন্ধ হবে: লুপ-অ্যান্টিল-ডান, ওয়ার্কট্রি আইসোলেশন এবং নন-ব্লকিং ব্যাকগ্রাউন্ড রান (দেখুন [`STRATEGY.md`](./STRATEGY.md))।

## ৩০-সেকেন্ড শুরু

**১. ইনস্টল** — একটি কমান্ড:

```bash
pi install npm:pi-taskflow
```

> **ঐচ্ছিক:** একবার `/tf init` চালিয়ে ১৮টি বিল্ট-ইন এজেন্টের মডেল রোল
> (`fast`, `strong`, `thinker`, …) আপনার নিজের মডেলে ম্যাপ করুন — একটি ইন্টারঅ্যাকটিভ পিকার।
> এটি স্কিপ করলে এজেন্টরা শুধু Pi-র ডিফল্ট মডেল ব্যবহার করে। দেখুন [মডেল রোল](#model-roles)।

**২. চালান** — শুধু Pi সেশনে মডেলকে বলুন:

> *একটি চেইন চালান: প্রথমে auth ফ্লো এক্সপ্লোর করুন, তারপর ফলাফলগুলি সারসংক্ষেপ করুন।*

মডেলটি `taskflow` টুলটি স্বয়ংক্রিয়ভাবে কল করে। আপনি লাইভ প্রোগ্রেস, প্রতি-ধাপ সময়, টোকেন খরচ এবং একটি সেভ করা রান রেকর্ড পাবেন — **বিল্ট-ইন টুলের সমান প্রচেষ্টা, এখন ট্র্যাকিং এবং রিজিউমযোগ্য।**

**৩. সেভ** — বলুন *"সেভ করো"* এবং আপনার কাছে `/tf:<name>` চিরকাল থাকবে।

এইটুকুই। আপনার কফি ঠান্ডা হওয়ার আগেই আপনি আপনার প্রথম ওয়ার্কফ্লো চালাতে পারবেন — একটি ফেজ ডেফিনিশন না লিখেই।

### শর্টহ্যান্ড (বিল্ট-ইন টুলের মতো একই আকার)

```jsonc
// একক — একটি এজেন্ট, একটি কাজ
{ "task": "src/-এর আর্কিটেকচার সারসংক্ষেপ করো", "agent": "explorer" }

// সমান্তরাল — একসাথে বেশ কয়েকটি চালান, আউটপুট একত্রিত হয়
{ "tasks": [
  { "task": "src/api-তে auth অডিট করো",             "agent": "analyst" },
  { "task": "src/api-তে ইনপুট ভ্যালিডেশন অডিট করো", "agent": "analyst" }
] }

// চেইন — অনুক্রমিক; প্রতিটি ধাপ পূর্ববর্তী আউটপুট দেখে
{ "chain": [
  { "task": "src/lib-এর পাবলিক API তালিকা করো", "agent": "scout" },
  { "task": "এর জন্য ডক লেখো:\n{previous.output}", "agent": "writer" }
] }
```

`agent` ঐচ্ছিক (প্রথম পাওয়া এজেন্ট ডিফল্ট)। রান লেবেল করতে এবং কমান্ড হিসেবে সেভ আনলক করতে `name` যোগ করুন।

## এটি চলতে দেখুন

এটি কোনো মকআপ নয়। **এটি একটি বাস্তব রানের stdout** — `self-improve` ফ্লো যা নিজস্ব টেস্ট স্যুট লিখে এবং যাচাই করে, একটি কোয়ালিটি গেটের মাঝখানে ধরা পড়েছে:

```
⊗ taskflow self-improve  6/7 · blocked · $0.095
    ✓ discover            agent   deepseek-v4-flash  10t ↑38k ↓6.7k $0.011
  ┌ ✓ write-runner-tests  agent   claude-sonnet-4-6  10t ↑13 ↓6.6k $0.020
  ├ ✓ write-store-tests   agent   claude-sonnet-4-6  10t ↑11 ↓10k $0.018
  ├ ✓ write-agents-tests  agent   claude-sonnet-4-6  10t ↑28 ↓13k $0.030
  └ ✓ fix-stability       agent   claude-sonnet-4-6  10t ↑13 ↓3.9k $0.012
    ✓ verify              gate    BLOCK 3 type errors in test files  deepseek-v4-flash
    ⊘ report              reduce  skipped · Gate blocked  ↳ fix-stability
```

**লেআউটটিই DAG।** কোনো ড্যাশবোর্ড নেই, গ্রেপ করার মতো লগ নেই — আপনি প্রোগ্রেস বার পড়েন এবং পুরো পাইপলাইন বুঝতে পারেন:

- **হেডার** — `⊗` = ব্লকড (একটি গেট থামিয়েছে); `6/7` ফেজ প্রসেসড; মোট খরচ `$0.095`।
- **স্ট্যাটাস আইকন** — `✓` ডান · `◐` চলছে · `✗` ব্যর্থ · `⊘` স্কিপড · `○` পেন্ডিং।
- **রেল `┌ ├ └`** — একই DAG লেয়ারের ফেজ, একসাথে চলছে। চারটি `write-*`/`fix-stability` টাস্ক `discover` থেকে ফ্যান আউট হয়েছে। ফাঁকা গটার = একক-ফেজ লেয়ার।
- **`↳`** — একটি লম্বা, লেয়ার-স্কিপিং ডিপেন্ডেন্সি। `report` নির্ভর করে সংলগ্ন `verify` *এবং* দুই লেয়ার পিছনে `fix-stability`-এর উপর, তাই শুধুমাত্র সেই স্কিপ এজটি অ্যানোটেটেড।
- **গেট** — `verify` `VERDICT: BLOCK` নির্গত করেছে, ফলে রানটাইম `report` স্কিপ করেছে এবং রান `blocked` হিসেবে শেষ করেছে, কারণটি ইনলাইনে দেখিয়েছে।
- **বিশদ** — প্রতি ফেজ: মডেল, টোকেন গণনা (`↑`ইন `↓`আউট), খরচ, সময়। ফ্যান-আউট ফেজগুলো সাব-টাস্ক প্রোগ্রেসও দেখায় (`3/15 2✗ 8▸`)।

## ডিক্লারেটিভ হোন

শর্টহ্যান্ড হল আপনার প্রবেশপথ। DSL-ই যেখানে `pi-taskflow` নিজের মূল্য প্রমাণ করে — ডায়নামিক ফ্যান-আউট, স্ট্রাকচার্ড রাউটিং এবং কোয়ালিটি গেট।

### ফ্যান আউট এবং রিডিউস

```jsonc
{
  "name": "summarize-files",
  "description": "ফাইল আবিষ্কার করো, প্রতিটি সারসংক্ষেপ করো, একটি রিপোর্ট তৈরি করো",
  "args": { "dir": { "default": "." } },
  "concurrency": 8,
  "phases": [
    { "id": "discover", "type": "agent", "agent": "scout",
      "task": "{args.dir}-এর অধীনে সোর্স ফাইল তালিকা করো (নন-রিকার্সিভ)।\nশুধুমাত্র একটি JSON অ্যারে আউটপুট দাও [{\"file\":\"\"}]। কোনো প্রোজ নয়।",
      "output": "json" },
    { "id": "summarize", "type": "map",
      "over": "{steps.discover.json}", "as": "item", "agent": "scout",
      "task": "{item.file} পড়ো এবং একটি এক-বাক্য সারসংক্ষেপ দাও।",
      "dependsOn": ["discover"] },
    { "id": "report", "type": "reduce", "from": ["summarize"], "agent": "writer",
      "task": "একটি সংক্ষিপ্ত ওভারভিউতে একত্রিত করো:\n{steps.summarize.output}",
      "dependsOn": ["summarize"], "final": true }
  ]
}
```

১. **`discover`** প্রতিটি ফাইল তালিকা করে এবং একটি JSON অ্যারে নির্গত করে।
২. **`summarize`** একটি `map` — এটি প্রতি ফাইলের জন্য একটি সাবএজেন্ট ফ্যান আউট করে, সর্বোচ্চ ৮টি কনকারেন্ট, `{item.file}` প্রতিটি পাথে বাউন্ড।
৩. **`report`** একটি `reduce` — এটি প্রতিটি সারসংক্ষেপকে একটি পরিষ্কার ওভারভিউতে একত্রিত করে।

মধ্যবর্তী সারসংক্ষেপগুলি কখনো আপনার কনটেক্সটে প্রবেশ করে না। রানটাইম সেগুলোর মালিক; আপনি রিপোর্টটি পান। **একবার সেভ করুন → `/tf:summarize-files dir=src` চিরকাল।**

### রুট, গেট, রিট্রাই, অ্যাপ্রুভ এবং খরচ সীমাবদ্ধ করুন

```jsonc
{
  "name": "triage-and-fix",
  "budget": { "maxUSD": 1.5 },
  "phases": [
    { "id": "triage", "type": "agent", "agent": "analyst", "output": "json",
      "task": "বাগটি শ্রেণীবদ্ধ করো। শুধুমাত্র {\"severity\":\"high\"} বা {\"severity\":\"low\"} আউটপুট দাও।" },
    { "id": "deep",  "when": "{steps.triage.json.severity} == high", "dependsOn": ["triage"],
      "agent": "executor-code", "task": "মূল কারণ বিশ্লেষণ করো এবং প্যাচ করো।",
      "retry": { "max": 2, "backoffMs": 500 } },
    { "id": "quick", "when": "{steps.triage.json.severity} == low",  "dependsOn": ["triage"],
      "agent": "executor-fast", "task": "দ্রুত ফিক্স প্রয়োগ করো।" },
    { "id": "approve", "type": "approval", "join": "any", "dependsOn": ["deep", "quick"],
      "task": "শিপ করার আগে ফিক্সটি রিভিউ করো।" },
    { "id": "ship", "type": "agent", "dependsOn": ["approve"],
      "task": "পরিবর্তন সহ একটি PR খোলো।", "final": true }
  ]
}
```

- **`when`** ট্রায়েজ JSON থেকে `deep` *বা* `quick`-এ রুট করে — অপর শাখাটি স্কিপ হয়।
- **`join: "any"`** `approve`-কে যেকোনো শাখা শেষ হওয়া মাত্র ফায়ার করতে দেয় (একটি OR-জয়েন)।
- **`retry`** ব্যাকঅফ সহ একটি অস্থির প্যাচ পুনরায় চালায়; **`budget`** পুরো রান থামায় যদি এটি খুব ব্যয়বহুল হয়ে যায়।
- **`approval`** চূড়ান্ত `ship`-এর আগে একজন মানুষের জন্য পজ করে (অ্যাপ্রুভ / রিজেক্ট / এডিট)।

কোনো স্ক্রিপ্টিং নেই। কোনো `eval` নেই। শুধু ডেটা যা রানটাইম এক্সিকিউট করে — LLM-জেনারেটেড ডেফিনিশন সরাসরি চালানোর জন্য যথেষ্ট নিরাপদ।

## ফেজ টাইপ

| টাইপ | কী করে | প্রয়োজনীয় ফিল্ড |
|------|--------------|-----------------|
| `agent` | একটি সাবএজেন্ট একটি একক টাস্ক চালায় | `task` |
| `parallel` | `branches[]` কনকারেন্টলি চালায় | `branches` (`{task, agent?}`-এর অ্যারে) |
| `map` | একটি অ্যারের উপর **ফ্যান আউট** — প্রতি আইটেমে একটি সাবএজেন্ট, `{item}` বাউন্ড | `over`, `task` |
| `gate` | কোয়ালিটি/রিভিউ ধাপ যা **ফ্লো থামাতে পারে** | `task` |
| `reduce` | `from[]` ফেজ আউটপুট একত্রিত করে একটি | `from`, `task` |
| `approval` | **হিউম্যান-ইন-দ্য-লুপ** পজ — অ্যাপ্রুভ / রিজেক্ট / এডিট | — |
| `flow` | একটি **সেভ করা সাব-ফ্লো** একটি ফেজ হিসেবে চালায় (কম্পোজিশন) | `use` |
| `loop` | **একটি টাস্ক শেষ না হওয়া পর্যন্ত পুনরাবৃত্তি করে** — একটি শর্ত, কনভার্জেন্স বা ক্যাপ না হওয়া পর্যন্ত বডি পুনরায় চালায় | `task`, `until` |
| `tournament` | **N ভ্যারিয়েন্ট প্রতিযোগিতা করে**, একজন জাজ সেরাটি বেছে নেয় (বা একত্রিত করে) | `task` \| `branches` |

### সাধারণ ফেজ ফিল্ড

প্রতিটি ফেজের একটি অনন্য `id` এবং একটি `type` প্রয়োজন (ডিফল্ট `agent`)। টাইপ-নির্দিষ্ট ফিল্ডের পাশাপাশি:

| ফিল্ড | অর্থ |
|---|---|
| `agent` | চালানোর জন্য এজেন্ট (প্রথম পাওয়া এজেন্ট ডিফল্ট) |
| `dependsOn` | ফেজ আইডি যেগুলোর জন্য এই ফেজ অপেক্ষা করে — DAG তৈরি করে |
| `join` | `"all"` (ডিফল্ট) প্রতিটি ডিপের জন্য অপেক্ষা করে; `"any"` একটি OR-জয়েন |
| `when` | কন্ডিশনাল গার্ড — এক্সপ্রেশন সত্য না হলে স্কিপ |
| `retry` | `{ max, backoffMs?, factor? }` — ব্যর্থ সাবএজেন্ট পুনরায় চেষ্টা |
| `output` | `"text"` (ডিফল্ট) বা `"json"` (`{steps.ID.json}` উন্মুক্ত করে) |
| `model` / `thinking` / `tools` | সাবএজেন্টের জন্য প্রতি-ফেজ ওভাররাইড |
| `cwd` | সাবএজেন্টের জন্য ওয়ার্কিং ডিরেক্টরি |
| `concurrency` | `map` / `parallel`-এর জন্য ফ্যান-আউট ক্যাপ (ফ্লো ডিফল্ট ওভাররাইড করে) |
| `final` | ফলাফল-বহনকারী ফেজ চিহ্নিত করে (অন্যথায় শেষ ফেজ জেতে) |
| `optional` | এখানে ব্যর্থতা রান **বাতিল করে না** |
| `use` / `with` | (`flow`) সেভ করা সাব-ফ্লোর নাম + তার আর্গুমেন্ট |
| `cache` | `{ scope, ttl?, fingerprint? }` — ক্রস-রান মেমোইজেশন (নিচে দেখুন) |

ফ্লো-লেভেল কী: `name`, `description`, `args`, `concurrency` (ডিফল্ট ৮), `agentScope` এবং `budget: { maxUSD?, maxTokens? }`।

### কন্ট্রোল ফ্লো ও রিলায়েবিলিটি

- **`when`** — একটি ফেজ স্কিপ করুন যদি না এক্সপ্রেশন সত্য হয়। `{refs}`, `== != < > <= >=`, `&& || !`, প্যারেনথেসিস এবং কোটেড স্ট্রিং/নাম্বার সাপোর্ট করে। জয়েন ফেজে `join: "any"`-এর সাথে যুক্ত করে বাস্তব if/else রাউটিং তৈরি করুন। পার্স ত্রুটি **ফেইল ওপেন**।
- **`join: "any"`** — একটি OR-জয়েন: ফেজটি *একটি* ডিপেন্ডেন্সি শেষ হওয়া মাত্র চলে (ডিফল্ট `"all"` সবগুলোর জন্য অপেক্ষা করে)।
- **`retry`** — `{ "max": 2, "backoffMs": 500, "factor": 2 }` ফিক্সড বা এক্সপোনেনশিয়াল ব্যাকঅফ সহ ব্যর্থ সাবএজেন্ট পুনরায় চালায়; ব্যবহার যোগ করা হয় এবং প্রচেষ্টা গণনা TUI-তে `↻N` হিসেবে দেখায়। ট্রানজিয়েন্ট প্রোভাইডার ত্রুটি (রেট-লিমিট / 5xx / টাইমআউট) **স্পষ্ট নীতি ছাড়াই অটো-রিট্রাই** করে; হার্ড ত্রুটিগুলি করে না।
- **`approval`** — একজন মানুষের জন্য পজ (অ্যাপ্রুভ / রিজেক্ট / এডিট)। রিজেক্ট ফ্লো থামায়; এডিট টাইপ করা নোটটি ফেজ আউটপুট হিসেবে ইনজেক্ট করে ডাউনস্ট্রিম ধাপের জন্য। নন-ইন্টারঅ্যাকটিভ রান অটো-অ্যাপ্রুভ করে।
- **`flow`** — `{ "type": "flow", "use": "deep-research", "with": { "topic": "{item}" } }` একটি সেভ করা ফ্লো একটি ফেজ হিসেবে চালায় (রিকারশন ডিটেক্ট এবং রিজেক্ট করা হয়)।

### লুপ-অ্যান্টিল-ডান (`loop`)

কিছু কাজ প্রকৃতিগতভাবে পুনরাবৃত্তিমূলক — একজন রিভিউয়ার সন্তুষ্ট না হওয়া পর্যন্ত একটি ড্রাফট পরিমার্জন করুন, টেস্ট পাস না হওয়া পর্যন্ত রিট্রাই-এন্ড-ইম্প্রুভ করুন, একটি উত্তরে কনভার্জ করুন। একটি `loop` ফেজ একটি স্টপ কন্ডিশন পূরণ না হওয়া পর্যন্ত একটি টাস্ক বডি পুনরায় চালায়:

```jsonc
{
  "id": "refine",
  "type": "loop",
  "task": "এই ড্রাফটটি উন্নত করো (ইটারেশন {loop.iteration})। পূর্ববর্তী প্রচেষ্টা:\n{loop.lastOutput}\n\nJSON ফেরত দাও {\"draft\":\"…\",\"done\":true|false}।",
  "until": "{steps.refine.json.done} == true",   // ইটারেশনের নিজস্ব আউটপুট এখানে উন্মুক্ত
  "output": "json",
  "maxIterations": 6,        // ডিফল্ট ১০, হার্ড ক্যাপ ১০০ — লুপ সর্বদা সমাপ্ত হয়
  "convergence": true        // ডিফল্ট: একটি ইটারেশনের আউটপুট শেষের সাথে অভিন্ন হলে তাড়াতাড়ি থামুন
}
```

- **বডি লোকাল** — টাস্ক `{loop.iteration}` (১-বেজড), `{loop.lastOutput}` (পূর্ববর্তী ইটারেশনের আউটপুট) এবং `{loop.maxIterations}` পড়তে পারে নিজের পূর্ববর্তী কাজের উপর নির্মাণ করতে; তিনটিই `until` কন্ডিশনের জন্যও উপলব্ধ।
- **`until`** — প্রতিটি ইটারেশনের পরে মূল্যায়ন করা হয় ইটারেশনের আউটপুট `{steps.<thisId>.output}` / `.json` হিসেবে উন্মুক্ত রেখে। `when`-এর মতো একই অপারেটর। এটি সত্য হওয়া মাত্র লুপ থামে।
- **সর্বদা সমাপ্ত হয়।** চারটি স্বাধীন স্টপ: `until` সত্য, **কনভার্জেন্স** (একটি ফিক্সড পয়েন্ট — আউটপুট পূর্ববর্তী ইটারেশনের সাথে অভিন্ন), **`maxIterations`** (হার্ড-ক্যাপড ১০০-এ), অথবা একটি **ব্যর্থ ইটারেশন** (ফেজ ব্যর্থ হয় আংশিক আউটপুট সংরক্ষিত রেখে)। একটি ত্রুটিপূর্ণ `until` চিরকাল ঘোরার পরিবর্তে **লুপ থামায়** (ফেইল-সেফ) এবং ফেজে একটি সতর্কতা দেখায়।
- TUI `↻N` দেখায় স্টপ কারণ সহ (`done` / `converged` / `max` / `failed`); ব্যবহার ইটারেশন জুড়ে যোগ করা হয়। `gate`/`approval`-এর মতো, `loop` **`cross-run` ক্যাশ থেকে বাদ** (প্রতিটি রানকে তাজা ইটারেট করতে হবে)।

### টুর্নামেন্ট (`tournament`)

ওপেন-এন্ডেড কাজের জন্য, সেরা ফলাফল প্রায়শই বেশ কয়েকটি ক্যান্ডিডেট জেনারেট করে এবং শক্তিশালীটি বেছে নেওয়ার মাধ্যমে আসে — একজন জাজ সহ বেস্ট-অব-N, একটি ডিক্লারেটিভ ফেজে:

```jsonc
{
  "id": "headline",
  "type": "tournament",
  "task": "এই লঞ্চ পোস্টের জন্য একটি আকর্ষণীয় হেডলাইন লেখো।",
  "variants": 4,                    // একই TASK-এর ৪টি প্রতিযোগী স্পন করুন (ডিফল্ট ৩, সর্বোচ্চ ২০)
  "judge": "সবচেয়ে শক্তিশালী হুক এবং পরিষ্কার প্রতিশ্রুতি আছে এমন হেডলাইন বাছাই করো।",
  "judgeAgent": "reviewer",          // ঐচ্ছিক; ফেজ এজেন্টে ডিফল্ট হয়
  "mode": "best"                     // "best" (ডিফল্ট) | "aggregate"
}
```

- **প্রতিযোগী** — হয় `variants: N` একটি `task`-এর কপি (মডেল ননডিটারমিনিজম থেকে বৈচিত্র্য আসে), অথবা স্বতন্ত্র `branches: [{task, agent?}, …]` যখন আপনি *ভিন্ন পদ্ধতি* পরস্পরের বিরুদ্ধে লড়াই করতে চান।
- **জাজ** — ফ্যান-আউটের পরে, একজন জাজ এজেন্ট প্রতিটি ভ্যারিয়েন্ট (নম্বরযুক্ত) আপনার `judge` রুব্রিক সহ দেখে এবং একটি `WINNER: <n>` লাইন বা `{"winner": n}` এর মাধ্যমে বিজয়ী বাছাই করে। একটি অপাঠ্য রায় **ফেইল ওপেন** করে ভ্যারিয়েন্ট ১-এ; ব্যর্থ জাজও ফিরে আসে — কাজ কখনো হারিয়ে যায় না।
- **`mode`** — `best` বিজয়ী ভ্যারিয়েন্ট **হুবহু** ফেরত দেয়; `aggregate` জাজের **সংশ্লেষিত** উত্তর ফেরত দেয় যা শক্তিশালী অংশগুলি একত্রিত করে।
- **শর্ট-সার্কিট:** যদি শুধুমাত্র একজন প্রতিযোগী বেঁচে থাকে, সে কোনো জাজ কল ছাড়াই জেতে; সব ব্যর্থ হলে ফেজ ব্যর্থ হয়। TUI `⚑ N→#k` দেখায়; ব্যবহার ভ্যারিয়েন্ট + জাজ যোগ করে। `gate`-এর মতো, এটি **`cross-run` ক্যাশ থেকে বাদ**।
- **`budget`** — একটি রান-ওয়াইড `{maxUSD, maxTokens}` সিলিং; একবার অতিক্রান্ত হলে, পেন্ডিং ফেজ স্কিপ হয় এবং চলমান ফ্যান-আউট স্পনিং বন্ধ করে, রান `blocked` হিসেবে শেষ করে।
- **আইডল ওয়াচডগ** — একটি সাবএজেন্ট যা ৫ মিনিটের জন্য নীরব থাকে তাকে হ্যাং ধরা হয় এবং কিল করা হয় (SIGTERM → SIGKILL), ফলে একটি হ্যাং চাইল্ড কখনো পুরো ফ্লো ফ্রিজ করতে পারে না।

### ক্রস-রান মেমোইজেশন (`cache`)

প্রতিটি ফেজ ইতিমধ্যেই কন্টেন্ট-অ্যাড্রেসড: একটি একক রানের **রিজিউম**-এর মধ্যে, একটি ফেজ যার রিজলভড ইনপুট অপরিবর্তিত তা স্কিপ করা হয়। `cache` সেই পুনর্ব্যবহার **স্বাধীন রান জুড়ে** প্রসারিত করে — যদি কোনো পূর্ববর্তী রান অভিন্ন ইনপুট হ্যাশ সহ একটি ফেজ গণনা করে থাকে, তার ফলাফল **$০.০০**-তে পুনর্ব্যবহৃত হয়।

```jsonc
{
  "id": "analyze-auth",
  "task": "auth মডিউল কীভাবে কাজ করে তা সারসংক্ষেপ করো।",
  "context": ["src/auth/**/*.ts"],
  "cache": {
    "scope": "cross-run",                 // "run-only" (ডিফল্ট) | "cross-run" | "off"
    "ttl": "6h",                          // ঐচ্ছিক সর্বোচ্চ বয়স; এর বেশি হলে হিট মিস ধরা হয়
    "fingerprint": ["git:HEAD", "glob:src/auth/**/*.ts"]  // বিশ্ব-অবস্থা কী-তে fold করুন
  }
}
```

- **`scope`** — `"run-only"` (ডিফল্ট) হল ঠিক ঐতিহাসিক আচরণ (শুধুমাত্র রানের মধ্যে রিজিউম)। `"cross-run"` ফেজটিকে পার্সিস্টেন্ট স্টোরে অপ্ট-ইন করে। `"off"` সম্পূর্ণরূপে পুনর্ব্যবহার অক্ষম করে (এমনকি রানের মধ্যে), ডিবাগিংয়ের জন্য।
- **তাজাতাই পুরো খেলা।** ক্যাশ কী ইতিমধ্যেই প্রম্পট, `over` আইটেম এবং যেকোনো `context` ফাইল (টাস্কে প্রি-রিড) অন্তর্ভুক্ত করে। `fingerprint` *ইমপ্লিসিট* ইনপুটকে কী-তে fold করে, যাতে "বিশ্ব বদলে গেছে" ক্যাশ মিস হয়: `git:HEAD`, `glob:<pat>` (সাইজ+mtime), `glob!:<pat>` (কন্টেন্ট হ্যাশ), `file:<path>`, `env:<NAME>`। `ttl` (`30m`/`6h`/`7d`) একটি সময় ব্যাকস্টপ।
- **সৎ সীমা:** একটি সাবএজেন্ট যে `context`/`fingerprint`-এ ঘোষিত নয় এমন একটি ফাইল পড়ে, তবুও একটি পুরানো `cross-run` হিট পরিবেশন করতে পারে। এজন্যই ডিফল্ট হল `run-only` এবং এজন্যই `gate`/`approval` ফেজগুলি `cross-run` থেকে **নিষিদ্ধ** (এগুলিকে প্রতিটি রানে তাজা ফলাফল দিতে হবে)। শুধুমাত্র সেই ফেজগুলির জন্যই অপ্ট-ইন করুন যাদের আউটপুট ঘোষিত ইনপুটের একটি ফাংশন।
- ক্যাশ `.pi/taskflows/cache/`-এ থাকে (গিটইগনোরড)। `action: "cache-clear"` দিয়ে পরিষ্কার করুন। পূর্ণ যুক্তি: [`docs/rfc-cross-run-memoization.md`](./docs/rfc-cross-run-memoization.md)।

### গেট ফেজ (কোয়ালিটি কন্ট্রোল)

একটি `gate` একটি এজেন্ট চালায় আপস্ট্রিম আউটপুট রিভিউ করতে এবং **বাকি ওয়ার্কফ্লো ব্লক করতে পারে।** গেট টাস্কটি শেষ করুন এমন একটি রায় চেয়ে যা রানটাইম পড়তে পারে:

- একটি চূড়ান্ত লাইন `VERDICT: PASS` বা `VERDICT: BLOCK` (এছাড়াও `OK`, `FAIL`, `STOP`, `REJECT`, `HALT` গ্রহণ করে — শেষ উপস্থিতি জেতে), অথবা
- JSON যেমন `{"continue": false, "reason": "missing auth checks"}` / `{"verdict": "block", "reason": "..."}`।

**BLOCK**-এ, ডাউনস্ট্রিম ফেজ স্কিপ হয় এবং রান `blocked` হিসেবে শেষ হয় কারণ উন্মুক্ত রেখে। **অস্পষ্ট আউটপুট ফেইল ওপেন** (PASS ধরা হয়) — একটি গেট কখনো দুর্ঘটনাক্রমে আপনার ফ্লো থামায় না।

```
নীচের অডিটটি রিভিউ করো। যদি কোনো এন্ডপয়েন্টে auth অনুপস্থিত থাকে,
"VERDICT: BLOCK" এবং এক-লাইন কারণ দিয়ে শেষ করো; অন্যথায় "VERDICT: PASS" দিয়ে শেষ করো।

{steps.audit.output}
```

## ইন্টারপোলেশন ও এক্সপ্রেশন

| প্লেসহোল্ডার | যা রিজলভ হয় |
|---|---|
| `{args.X}` | ইনভোকেশন আর্গুমেন্ট |
| `{steps.ID.output}` | পূর্ববর্তী ফেজের টেক্সট আউটপুট |
| `{steps.ID.json}` | পূর্ববর্তী আউটপুট JSON হিসেবে পার্সড (বা `{steps.ID.json.field}`) |
| `{item}` / `{item.field}` | `map` ফেজের মধ্যে বর্তমান আইটেম |
| `{previous.output}` | তাৎক্ষণিকভাবে আপস্ট্রিম ফেজ আউটপুট |

কন্ডিশন গ্রামার (`when`-এর জন্য): `== != < > <= >=`, `&& || !`, প্যারেনথেসিস, কোটেড স্ট্রিং/নাম্বার এবং যেকোনো `{...}` রেফারেন্স — যেমন `"when": "{steps.triage.json.route} == deep && {args.force} != true"`।

> `dependsOn`-এ ঘোষিত নয় এমন `{steps.X}` রেফারেন্স করা একটি **কঠোর ভ্যালিডেশন ত্রুটি** — একটি এজেন্ট চালানোর আগেই রানটাইম সবচেয়ে সাধারণ পাইপলাইন বাগ ধরে ফেলে।

## কমান্ড

সেভ করা ফ্লো CLI শর্টকাট হয়ে যায়। সব কমান্ড Pi সেশনে চলে:

| কমান্ড | কী করে |
|---|---|
| `/tf list` | সব সেভ করা ফ্লো তালিকা করো |
| `/tf run <name> [args]` | একটি সেভ করা ফ্লো চালাও (যেমন `/tf run summarize-files dir=src`) |
| `/tf show <name>` | একটি ফ্লোর ডেফিনিশন প্রিন্ট করো |
| `/tf runs` | সাম্প্রতিক রান ইতিহাস ব্রাউজ করো (ইন্টারঅ্যাকটিভ TUI) |
| `/tf resume <runId>` | একটি পজ করা/ব্যর্থ রান চালিয়ে যাও — ক্যাশেড ফেজ স্বয়ংক্রিয়ভাবে স্কিপ |
| `/tf init` | **ইন্টারঅ্যাকটিভলি মডেল রোল ম্যাপ করো** আপনার সক্রিয় মডেলের সাথে (`~/.pi/agent/settings.json` লেখে) |
| `/tf:<name> [args]` | শর্টকাট — এক ট্যাপে ফ্লো চালায় |

টুল অ্যাকশন (মডেল ব্যবহার করে): `run` (ইনলাইন `define` বা সেভ করা `name`), `save`, `resume`, `list`, `init`।

## সেশন জুড়ে রিজিউম

একটি টাস্কফ্লো রান আপনার সেশনের সাথে আবদ্ধ নয়। প্রতিটি সম্পূর্ণ ফেজ ডিস্কে লেখা হয়, ফলে যে রান ব্যর্থ হয় (অথবা আপনি থামান) তা পরে `/tf resume <runId>` দিয়ে চালিয়ে যাওয়া যেতে পারে — **ক্যাশেড ফেজ স্বয়ংক্রিয়ভাবে স্কিপ করে** এবং শুধুমাত্র বাকি কাজ টোকেন খরচ করে।

## স্টোরেজ

```
.pi/taskflows/<name>.json          # প্রজেক্ট-স্কোপড ডেফিনিশন (শেয়ার করতে কমিট করুন)

.pi/taskflows/runs/<runId>.json    # রিজিউমের জন্য রান স্টেট (এটি গিটইগনোর করুন)
```

> `.pi/taskflows/` কমিট করুন এবং আপনার পুরো টিম পাইপলাইন শেয়ার করে — কোনো কনফিগ সিঙ্ক নেই, কোনো অনবোর্ডিং ডক নেই। রান স্টেট অ্যাটোমিক্যালি লেখা হয় এবং একটি জিরো-ডিপেন্ডেন্সি ফাইল লক দ্বারা সুরক্ষিত, ফলে কনকারেন্ট রান কখনো ইনডেক্স দূষিত করে না।

এজেন্ট ডিসকভারি স্কোপ (ফ্লো ডেফিনিশনে `agentScope` এর মাধ্যমে):

| মান | এজেন্ট আবিষ্কার করে |
|---|---|
| `"user"` (ডিফল্ট) | `~/.pi/agent/agents/*.md` থেকে |
| `"project"` | `.pi/agents/*.md` থেকে (ট্রি উপরে ওয়াক করে) |
| `"both"` | ইউজার + প্রজেক্ট; নাম সংঘর্ষে প্রজেক্ট জেতে |

## এজেন্ট

টাস্কফ্লোতে **১৮টি বিল্ট-ইন এজেন্ট** রয়েছে — প্রতিটি একটি `.md` ফাইল যার একটি টিউনড সিস্টেম প্রম্পট, থিংকিং লেভেল এবং টুল সেট। ইনস্টলের পরপরই যেকোনো ফেজ বা শর্টহ্যান্ডে এদের `name` দ্বারা রেফারেন্স করতে পারেন। কোনো সেটআপ প্রয়োজন নেই।

### মডেল রোল

প্রতিটি বিল্ট-ইন এজেন্টের `model` ফিল্ড একটি **রোল প্লেসহোল্ডার** ব্যবহার করে (যেমন `{{fast}}`) শক্তভাবে লেখা প্রোভাইডার স্ট্রিংয়ের পরিবর্তে। এটি *ইনটেন্ট* কে *ইমপ্লিমেন্টেশন* থেকে বিচ্ছিন্ন করে — আপনি একবার মডেলের সাথে রোল ম্যাপ করেন, এবং প্রতিটি এজেন্ট অ্যাডাপ্ট করে।

| রোল | ইনটেন্ট | সাধারণ মডেল |
|---|---|---|
| `{{fast}}` | সস্তা ও দ্রুত — উচ্চ-ভলিউম, কম-ঝুঁকি | DeepSeek V4 Flash |
| `{{strong}}` | সুষম — প্ল্যানিং, রিভিউ, মাঝারি জটিলতা | MiMo v2.5 Pro |
| `{{thinker}}` | গভীর বিশ্লেষণ — রিকোয়ারমেন্ট, সমালোচনা | DeepSeek V4 Pro |
| `{{arbiter}}` | চূড়ান্ত রায় — টাইব্রেক, প্ল্যান কোয়ালিটি গেট | Qwen 3.7 Max |
| `{{vision}}` | মাল্টিমোডাল — UI কাজ, ডিজাইন পড়া | MiniMax M3 |
| `{{reasoner}}` | সতর্ক যুক্তি — সিকিউরিটি, ঝুঁকি | GLM 5.1 |

কনফিগারেশন ছাড়া, এজেন্টরা Pi-র ডিফল্ট মডেলে ফিরে যায়। বাস্তব মডেলে রোল ম্যাপ করতে, ইন্টারঅ্যাকটিভ সেটআপ চালান:

```bash
/tf init
```

`/tf init` একটি **অ্যাকশন মেনু** দিয়ে শুরু হয়। প্রথমবার ব্যবহারকারীরা একটি ২-অপশন শর্টকাট পান ("Use recommended defaults" / "Configure each role")। ফিরে আসা ব্যবহারকারীরা সম্পূর্ণ ৫-অপশন মেনু দেখেন:

```
? মডেল রোলের সাথে কী করতে চান?
  ❯ Use recommended defaults
    Configure each role
    Edit one role
    Show current roles
    Cancel
```

পিকারটি মডেলের **ডিসপ্লে নাম** ক্যাপাবিলিটি ফ্ল্যাগ এবং বর্তমান/সুপারিশকৃত মার্কার সহ দেখায়:

### টুল পাথ (`action="init"`)

মডেল `taskflow` টুলের মাধ্যমেও রোল কনফিগার করতে পারে:

| মোড | আচরণ |
|---|---|
| `mode: "show"` (ডিফল্ট) | বর্তমান `modelRoles`-এর রিড-অনলি রিপোর্ট। কখনো ওভাররাইট করে না। |
| `mode: "apply-defaults"` + `force: true` | `RECOMMENDED_DEFAULTS`-কে `settings.json`-এ লেখে, পুরনো কী সংরক্ষণ করে। |
| `mode: "interactive"` | সম্পূর্ণ অ্যাকশন মেনু + পিকার ফ্লো চালু করে (একটি UI সেশন প্রয়োজন)। |

> **v0.0.13 ডিপ্রিকেশন নোট:** যদি `mode` বাদ দেওয়া হয়, টুলটি v0.0.12 আচরণে ফিরে যায় যখন `modelRoles` খালি থাকে (অটো-লেখে ডিফল্ট) একটি `console.warn` ডিপ্রিকেশন নোটিশ সহ। যদি `modelRoles` ইতিমধ্যেই থাকে, এটি `mode: "show"`-এর মতো আচরণ করে। এই ব্রিজটি v0.0.14-তে সরানো হবে।

### কাস্টম এজেন্ট

`~/.pi/agent/agents/` (ইউজার-লেভেল) বা `.pi/agents/` (প্রজেক্ট-লেভেল, এটি কমিট করুন)-এ একটি `.md` ফাইল রাখুন আপনার নিজস্ব এজেন্ট যোগ করতে:

```markdown
---
name: my-linter

description: ESLint চালান এবং লঙ্ঘন রিপোর্ট করুন

tools: read, bash

model: "{{fast}}"

thinking: off
---

আপনি একটি লিন্টিং এজেন্ট। প্রদত্ত ফাইলগুলিতে `npx eslint --format json` চালান।
ফাইল অনুযায়ী গ্রুপ করে লঙ্ঘন রিপোর্ট করুন। কোনো ফিক্স নয়।
```

তারপর যেকোনো ফেজে এটি রেফারেন্স করুন: `{ "agent": "my-linter", "task": "Lint src/" }`।

## 🍽️ আমরা নিজেদের ডগফুড খাই

`pi-taskflow`-এর প্রতিটি ফিচার **`pi-taskflow`-এর মাধ্যমেই** শিপ হয়।

## স্ট্যাটাস ও সীমাবদ্ধতা

**v0.0.13** — লুপ-আন্টিল-ডান (`loop` ফেজ: একটি শর্ত, কনভার্জেন্স বা ক্যাপ পর্যন্ত ইটারেট), টুর্নামেন্ট (একজন জাজ সহ বেস্ট-অব-N), ক্রস-রান মেমোইজেশন (গিট/ফাইল/গ্লোব/এনভ ফিঙ্গারপ্রিন্ট এবং TTL সহ কন্টেন্ট-অ্যাড্রেসড ক্যাশ), ইন্টারঅ্যাকটিভ `/tf init` রোল-অ্যাওয়্যার মডেল পিকার + ডিফ প্রিভিউ + অ্যাটমিক মার্জ-রাইট সহ, ১৮টি বিল্ট-ইন এজেন্ট ৬টি মডেল রোল সহ। DSL + DAG রানটাইমের (`agent`/`parallel`/`map`/`gate`/`reduce`) উপরে সম্পূর্ণ কন্ট্রোল-ফ্লো ও রিলায়েবিলিটি লেয়ার (`when` গার্ড, `join: any`, `retry`/ব্যাকঅফ, `approval`, `flow` কম্পোজিশন, `budget` ক্যাপ, আইডল ওয়াচডগ)। ইনলাইন + সেভ করা ফ্লো, ক্রস-সেশন রিজিউম, লাইভ প্রোগ্রেস এবং আইসোলেটেড কনটেক্সট। একটি রান একটি একক স্ট্রিমিং টুল কল হিসেবে এক্সিকিউট হয়।

পরিচিত সীমানা (ট্র্যাক করা, সীমাবদ্ধ — মাঝপথে কোনো চমক নেই):

- **কোনো ডিটাচড ব্যাকগ্রাউন্ড এক্সিকিউশন নেই।** একটি রানের জন্য Pi সেশন খোলা থাকা প্রয়োজন। প্রকৃত ব্যাকগ্রাউন্ড এক্সিকিউশন (এবং এর উপরে ইভেন্ট/ক্রন ট্রিগার) রোডম্যাপে আছে।
- **কোনো `output: "file"` নেই।** আউটপুট শুধুমাত্র টেক্সট/JSON — একটি এজেন্টের `write` টুল কলের মাধ্যমে ফাইল লিখুন।
- **`map`-এর জন্য একটি JSON অ্যারে প্রয়োজন।** `over` ফিল্ডটি অবশ্যই একটি `{steps.ID.json}` অ্যারেতে রিজলভ হতে হবে। প্রথমে একটি একক-এজেন্ট `output: "json"` ফেজে একটি টেক্সট তালিকা মোড়ান।
- **DAG অবশ্যই অ্যাসাইক্লিক হতে হবে।** চক্র ভ্যালিডেশনে প্রত্যাখ্যাত হয়।
