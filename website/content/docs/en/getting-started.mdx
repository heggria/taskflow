# Getting Started

## On Pi

Install the extension:

```bash
pi install npm:pi-taskflow
```

Then in a Pi session, describe the work:

> Run a chain: first explore the auth flow, then summarize the findings.

The model calls the `taskflow` tool automatically. You get live progress, per-step timing, token cost, and a saved run record.

Say **"save it"** and you have `/tf:<name>` forever.

### Optional setup

Run `/tf init` once to map the 18 built-in agents' model roles (`fast`, `strong`, `thinker`, …) to your own models. Skip it and agents just use Pi's default model.

## On Codex

Install the plugin:

```bash
codex plugin marketplace add heggria/taskflow
codex plugin add taskflow@taskflow
```

Then ask Codex to run a multi-phase job:

> Run a taskflow that audits every route in src/routes and reports missing auth.

Codex calls the `taskflow_*` MCP tools automatically.

## The shorthand

You already know your agent's built-in subagent shorthand — `task` / `tasks` / `chain`. taskflow speaks the same shape:

```json
{
  "task": "Summarize the architecture of src/",
  "agent": "explorer"
}
```

```json
{
  "tasks": [
    { "task": "Audit auth in src/api", "agent": "analyst" },
    { "task": "Audit input validation in src/api", "agent": "analyst" }
  ]
}
```

```json
{
  "chain": [
    { "task": "List the public API of src/lib", "agent": "scout" },
    { "task": "Write docs for:\n{previous.output}", "agent": "writer" }
  ]
}
```

`agent` is optional (defaults to the first discovered agent). Add a `name` to label the run and unlock saving it as a command.

## Go declarative

When you outgrow the shorthand, the full DSL gives you a real DAG:

```json
{
  "name": "summarize-files",
  "description": "Discover files, summarize each, produce one report",
  "args": { "dir": { "default": "." } },
  "concurrency": 8,
  "phases": [
    {
      "id": "discover",
      "type": "agent",
      "agent": "scout",
      "task": "List source files under {args.dir} (non-recursive).\nOutput ONLY a JSON array [{\"file\":\"\"}]. No prose.",
      "output": "json"
    },
    {
      "id": "summarize",
      "type": "map",
      "over": "{steps.discover.json}",
      "as": "item",
      "agent": "scout",
      "task": "Read {item.file} and give a one-sentence summary.",
      "dependsOn": ["discover"]
    },
    {
      "id": "report",
      "type": "reduce",
      "from": ["summarize"],
      "agent": "writer",
      "task": "Combine into a short overview:\n{steps.summarize.output}",
      "dependsOn": ["summarize"],
      "final": true
    }
  ]
}
```

How it runs:

1. `discover` lists files and emits JSON.
2. `summarize` fans out one subagent per file, throttled to 8 concurrent.
3. `report` merges every summary into one clean overview.

Only the report reaches your context.

## Verify before running

For any non-trivial flow, verify it first:

```bash
/tf verify <name>
```

This catches cycles, missing deps, undefined refs, and contract typos — zero tokens.
