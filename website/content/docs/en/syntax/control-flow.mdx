# Control Flow

<Callout type="info">
  taskflow provides several primitives for routing and reliability.
</Callout>

## `when` ‚Äî conditional guards

Skip a phase unless the expression is truthy.

```json title="when-guard-example.json"
{
  "id": "deep",
  "when": "{steps.triage.json.severity} == high",
  "dependsOn": ["triage"]
}
```

Operators: `==`, `!=`, `<`, `>`, `<=`, `>=`, `&&`, `||`, `!`, parentheses, quoted strings/numbers.

<Callout type="warn">
  **Fail-open:** a parse error in `when` causes the phase to run, never silently drop.
</Callout>

## `join` ‚Äî dependency joins

- `"all"` (default) ‚Äî wait for every dependency.
- `"any"` ‚Äî run as soon as one dependency completes.

Use `join: "any"` with conditional branches to build if/else routing:

```json title="join-any-routing-example.json"
{
  "id": "triage",
  "type": "agent",
  "output": "json",
  "task": "Classify severity as high or low."
},
{
  "id": "deep",
  "when": "{steps.triage.json.severity} == high",
  "dependsOn": ["triage"]
},
{
  "id": "quick",
  "when": "{steps.triage.json.severity} == low",
  "dependsOn": ["triage"]
},
{
  "id": "merge",
  "type": "reduce",
  "from": ["deep", "quick"],
  "join": "any",
  "dependsOn": ["deep", "quick"]
}
```

## `retry`

Retry a failing phase:

```json title="retry-example.json"
{
  "id": "patch",
  "retry": {
    "max": 3,
    "backoffMs": 500,
    "factor": 2
  }
}
```

- `max` ‚Äî max attempts.
- `backoffMs` ‚Äî initial delay.
- `factor` ‚Äî `1` = fixed, `2` = exponential.

Transient provider errors (rate limits, 5xx, timeouts) are auto-retried even without an explicit policy.

## `optional`

A failure in an `optional` phase does not abort the run.

```json title="optional-example.json"
{
  "id": "lint",
  "type": "script",
  "run": "npm run lint",
  "optional": true
}
```

Pair with a fallback phase guarded by `when`.

## `onBlock` ‚Äî self-healing gates

By default a blocking gate halts the run. With `"retry"`, the gate re-runs its upstream `dependsOn` phases and re-evaluates, up to `retry.max` rounds.

```json title="onBlock-retry-example.json"
{
  "id": "quality",
  "type": "gate",
  "onBlock": "retry",
  "retry": { "max": 3 },
  "dependsOn": ["implement"],
  "task": "Does this satisfy all criteria? VERDICT: PASS or BLOCK."
}
```

This creates a generate‚Üícritique‚Üíregenerate rework loop.

## `idempotent`

Set `idempotent: false` for phases with irreversible side effects (webhook POSTs, deploys, DB writes). This:

- Suppresses implicit transient auto-retry.
- Excludes the result from all caches.
- Records `sideEffect: true` in the phase state.
