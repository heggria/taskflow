# Tournament Selection

<Callout type="info">
  For open-ended creative or subjective work, generate several candidates and let a judge pick the strongest.
</Callout>

## Basic tournament

## Basic tournament

```json title="tournament-best-example.json"
{
  "id": "headline",
  "type": "tournament",
  "task": "Write a punchy headline for this launch post.",
  "variants": 4,
  "judge": "Pick the headline with the strongest hook and clearest promise.",
  "mode": "best"
}
```

## How it works

<Steps>
  <Step>Spawn `variants` copies of the task.</Step>
  <Step>Run a judge agent that sees every variant.</Step>
  <Step>The judge emits `WINNER: <n>` or JSON `{"winner": n}`.</Step>
  <Step>Return the winning variant (`mode: "best"`) or a synthesized answer (`mode: "aggregate"`).</Step>
</Steps>

## Branches

For genuinely different approaches, use `branches`:

```json title="tournament-branches-example.json"
{
  "id": "headline",
  "type": "tournament",
  "branches": [
    { "task": "Write a bold, provocative headline." },
    { "task": "Write a clear, benefit-driven headline." },
    { "task": "Write a question-based headline." }
  ],
  "judge": "Pick the best headline.",
  "mode": "best"
}
```

## Fail-open

<Callout type="warn">
  If the judge's verdict is unparseable, the tournament falls back to variant 1. Work is never lost.
</Callout>

## When to use

<Callout type="tip">
  - Creative writing where quality is subjective.
  - Open-ended summarization.
  - Naming, branding, or headline generation.
  - Any task where one shot is unreliable.
</Callout>
