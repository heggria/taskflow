# Introduction

**taskflow** is a declarative, verifiable _graph of tasks_ for coding-agent subagents.

Not a workflow you script — a **DAG you declare**. Fan out · gate · loop · tournament · resume · save as a command — intermediate results stay out of your context.

Runs on the [Pi](https://pi.dev) coding agent and on [OpenAI Codex](https://github.com/openai/codex).

## The problem with raw subagents

When you delegate multi-step work to a coding agent, you usually describe the plan in prose. The model then:

1. Re-derives the plan every run.
2. Floods your context with intermediate transcripts.
3. Starts over from zero when one call fails.
4. Gives you no way to _check_ the plan before it burns tokens.

There is no reuse, no recovery, no structure.

## The taskflow answer

`taskflow` moves the plan **out of the prompt and into a declarative graph of task nodes.** The runtime owns the DAG, the loops, the retries, and the intermediate state. You declare a pipeline once and run it a hundred times — by name.

Because the plan is data, not prose and not code, it can be **validated, visualized, and replayed.**

## What makes it different

| | subagent (built-in) | **taskflow** |
|---|---|---|
| **Who drives** | the model, turn by turn | the runtime, from a definition |
| **Topology** | chain / flat parallel | **DAG with layered concurrency + routing** |
| **Intermediate results** | in your context window | **in the runtime — not your context** |
| **Scale** | a handful of tasks | **dynamic `map` fan-out over dozens of items** |
| **Reusable** | re-described every time | **saved by name** |
| **Resumable** | ✗ | **✓ cross-session** |
| **Quality gates** | ✗ | **`gate` phases** |
| **Conditional routing** | ✗ | **`when` guards + `join: any`** |
| **Fault tolerance** | ✗ | **per-phase `retry`** |
| **Human-in-the-loop** | ✗ | **`approval` phases** |
| **Cost control** | ✗ | **run-wide `budget`** |
| **Composition** | ✗ | **`flow` sub-flows** |
| **Iterative loops** | ✗ | **`loop` phases** |
| **Competitive selection** | ✗ | **`tournament` phases** |
