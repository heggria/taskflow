# 在 Codex 上使用 taskflow

taskflow 以**插件**形式运行在 [Codex](https://github.com/openai/codex) 上。它通过 MCP 服务器暴露 `taskflow_*` 工具，并附带一个路由 skill，告诉 Codex 何时调用这些工具。

## 安装

```bash
codex plugin marketplace add heggria/taskflow
codex plugin add taskflow@taskflow
```

无需手动 `mcp add`，无需编辑配置。插件通过 `npx` 声明 MCP 服务器，因此服务器会按需获取并启动。

验证：

```bash
codex plugin list   # taskflow@taskflow 已安装、已启用
codex mcp list      # taskflow 已启用
```

## 暴露的工具

| 工具 | 用途 |
|---|---|
| `taskflow_run` | 运行已保存的 flow 或内联定义。只返回最终阶段输出。 |
| `taskflow_list` | 列出从当前工作目录可发现的已保存 flow。 |
| `taskflow_show` | 以 JSON 形式展示已保存 flow 的定义。 |
| `taskflow_verify` | 静态验证 flow，不执行。 |
| `taskflow_compile` | 将 flow 的 DAG 渲染为图表 + 文本大纲。 |

## 使用

直接描述需求即可——内置 skill 会告诉 Codex 何时调用工具：

```
> 列出我保存的 taskflow。
> 验证这个 flow：{name:"x", phases:[{id:"a", type:"agent", agent:"writer", task:"draft"}]}
> 运行 "release-train" taskflow。
```

## 长时间运行的 flow

`taskflow_run` 会在整个 DAG 完成后才返回。Codex 对每个 MCP 服务器有工具调用超时；插件默认设置为 30 分钟。

对于非常大的 flow，建议拆成几个较小的 `taskflow_run` 调用，使每个都能在超时窗口内返回。

## 卸载

```bash
codex plugin remove taskflow@taskflow
```
