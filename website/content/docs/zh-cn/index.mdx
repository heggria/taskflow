# 简介

**taskflow** 是面向编程智能体子代理的声明式、可验证的*任务图*。

不是脚本化的 workflow，而是声明式的 **DAG**。支持 fan-out、gate、loop、tournament、断点续跑，并保存为命令——中间结果不会进入你的上下文。

支持 [Pi](https://pi.dev) 编程智能体和 [OpenAI Codex](https://github.com/openai/codex)。

## 原始子代理的问题

当你把多步骤工作委托给编程智能体时，通常会用自然语言描述计划。然后模型会：

1. 每次运行都重新推导计划。
2. 用中间过程填满你的上下文窗口。
3. 一旦某一步失败就从头开始。
4. 无法在消耗 token 之前*检查*计划。

没有复用、没有恢复、没有结构。

## taskflow 的答案

`taskflow` 把计划**从 prompt 中移出，变成声明式的任务节点图。**运行时拥有 DAG、循环、重试和中间状态。你声明一次流水线，就可以运行一百次——按名称调用。

因为计划是数据，而不是自然语言或代码，它可以被**验证、可视化和重放。**

## 不同之处

| | 内置子代理 | **taskflow** |
|---|---|---|
| **驱动方** | 模型逐轮驱动 | 运行时按定义驱动 |
| **拓扑** | 链式 / 扁平并行 | **带分层并发和路由的 DAG** |
| **中间结果** | 在你的上下文窗口里 | **在运行时里，不在上下文中** |
| **规模** | 少量任务 | **动态 map fan-out 数十个条目** |
| **可复用** | 每次重新描述 | **按名称保存** |
| **可续跑** | ✗ | **✓ 跨会话** |
| **质量门** | ✗ | **`gate` 阶段** |
| **条件路由** | ✗ | **`when` 守卫 + `join: any`** |
| **容错** | ✗ | **每阶段 `retry`** |
| **人工介入** | ✗ | **`approval` 阶段** |
| **成本控制** | ✗ | **运行级 `budget`** |
| **组合** | ✗ | **`flow` 子流程** |
| **迭代循环** | ✗ | **`loop` 阶段** |
| **竞争选择** | ✗ | **`tournament` 阶段** |

## 快速安装

### Pi

```bash
pi install npm:pi-taskflow
```

### Codex

```bash
codex plugin marketplace add heggria/taskflow
codex plugin add taskflow@taskflow
```
