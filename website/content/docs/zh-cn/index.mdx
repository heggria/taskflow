欢迎来到 **taskflow** 文档。taskflow 是面向编程智能体子代理的声明式、可验证的*任务图*——你把多步骤的智能体工作描述成一个 DAG，运行时会在花费一个 token 之前先验证它，并且只有最终结果会进入你的上下文窗口。

整个想法用四行 JSON 就能说清：

```json title="一个完整的 taskflow"
{
  "task": "Summarize the architecture of src/",
  "agent": "explorer"
}
```

这就是一个完整的 taskflow。运行时把它变成一张图，跑一个子代理，然后返回结果。再加上更多阶段，用 `dependsOn` 把它们连起来，同样的结构就能扩展到对上百个文件做 fan-out——重试、gate、loop、跨会话续跑都替你处理好。

## 它是为谁准备的？

taskflow 是为那些把多步骤工作委托给编程智能体、并且遇到以下情况的人准备的：

- **上下文窗口压力。** 每个子代理的中间记录都涌入你的对话，一次五步审计还没看到结果就把上下文吃光了。
- **计划只存在于运行时。** 你用自然语言描述一个工作流，模型每次运行都重新推导一遍，既没有东西可以 diff、复用，也没有东西可以交给评审者。
- **没有恢复能力。** 一次运行死在最后一步，你只能从头再来——每个子代理调用都得重新付费。
- **可复用的流水线。** 你想给一个工作流命名一次，然后按名称调用它，无论在 Pi、Codex、Claude Code、OpenCode 还是 Grok Build 上。

如果其中任何一条听起来耳熟，taskflow 就是为你做的。
