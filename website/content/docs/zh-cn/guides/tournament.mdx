# 锦标赛选择

<Callout type="info">
  对于开放式创意或主观工作，生成多个候选并让评委选出最强的。
</Callout>

## 基本锦标赛

```json title="锦标赛最佳示例.json"
{
  "id": "headline",
  "type": "tournament",
  "task": "为这篇发布帖子写一个醒目的标题。",
  "variants": 4,
  "judge": "选择最有吸引力、承诺最清晰的标题。",
  "mode": "best"
}
```

## 工作原理

<Steps>
  <Step>生成 `variants` 份任务副本。</Step>
  <Step>运行评委 agent，它看到每个变体。</Step>
  <Step>评委输出 `WINNER: <n>` 或 JSON `{"winner": n}`。</Step>
  <Step>返回获胜变体（`mode: "best"`）或综合答案（`mode: "aggregate"`）。</Step>
</Steps>

## Branches

对于真正不同的方法，使用 `branches`：

```json title="锦标赛分支示例.json"
{
  "id": "headline",
  "type": "tournament",
  "branches": [
    { "task": "写一个大胆、挑衅的标题。" },
    { "task": "写一个清晰、利益驱动的标题。" },
    { "task": "写一个基于问题的标题。" }
  ],
  "judge": "选择最佳标题。",
  "mode": "best"
}
```

## Fail-open

<Callout type="warn">
  如果评委裁决无法解析，锦标赛回退到变体 1。不会丢失工作。
</Callout>

## 何时使用

<Callout type="tip">
  - 质量主观的创意写作。
  - 开放式摘要。
  - 命名、品牌或标题生成。
  - 任何一次尝试不可靠的任务。
</Callout>
