# 指南

指南是这些文档中*实用*的那一半。[核心概念](/zh-cn/docs/concepts)解释模型，[语法参考](/zh-cn/docs/syntax)列举每个字段，而指南带你端到端地做一件真实的事——在某个宿主上安装、跑一个 flow、保存它、调试它，并在工作需要时上手一个进阶模式。

它们分两种：

- **按宿主。** taskflow 跑在 Pi、Codex、Claude Code 和 OpenCode 上。每个宿主有自己的安装路径和调用入口，所以各自有专门的走查。
- **按模式。** 基础跑通之后，有些问题想要特定的形状——一个在运行时规划自己子流程的模型，或者一个生成多份竞争草稿再由评委挑出最佳的锦标赛。

<Callout type="tip">
  刚接触 taskflow？从你已经在用的宿主开始。两个宿主指南都以同一个可运行的 review flow 收尾，所以之后你可以跨宿主对照着看。
</Callout>
