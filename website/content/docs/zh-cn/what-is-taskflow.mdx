# 什么是 taskflow？

**taskflow** 是面向编程智能体子代理的声明式、可验证的*任务图*。

简而言之：它让你把多步骤智能体工作描述成一个由离散**任务**节点组成的 **DAG**（有向无环图），在任何一个模型调用运行之前就验证该图，然后以动态 fan-out、gate、loop、tournament 和跨会话续跑的方式执行它。

## 为什么叫 "taskflow" 而不是 "workflow"？

名字本身就是论点。

在工程领域，**task** 是*离散的、声明的工作单元*——任务图的节点（与构建系统、调度器或编译器接入 DAG 的 `task` 是同一个概念）。而 **work** 是*流动且无界的*——连续的命令式执行过程。

这正是当前编程智能体生态系统中的设计分歧：

- **`workflow`**（动态代码模式）是模型写出的**命令式脚本**：`await agent(...)`、`if`、`for`、又一个 `await`。它很表达力强——图灵完备——但图只有在代码运行时才存在。你无法在看到它、diff 它或在付费之前证明它会终止。
- **`taskflow`** 把计划**从代码中移出，变成声明式的 `task` 节点图。**因为图是*数据*，运行时能做命令式脚本结构上做不了的事：**静态验证**（无循环、无死胡同、无预算溢出、无悬空引用）后再生成任何子代理，**渲染它**、**按阶段续跑它**，并把它保存为一个单词的命令。

> **我们故意做出的权衡：** 我们放弃任意代码的原始表达力，以换取命令式脚本无法拥有的东西——一个**可验证、可观察、可重放、且可安全由 LLM 生成**的图。

## 核心理念

### 1. 计划是数据

taskflow 定义是 JSON（或 YAML、或 MDX frontmatter）。它声明：

- **阶段（Phases）** — 离散任务节点（`agent`、`map`、`gate`、`reduce`、`loop`、`tournament`、`approval`、`flow`、`script`）。
- **边（Edges）** — `dependsOn` 关系，构成 DAG。
- **控制流** — `when` 守卫、`join: any/all`、`retry`、`budget`。

因为它是数据，运行时才能验证它、可视化它、缓存它的结果。

### 2. 在消耗 token 之前静态验证

执行前，运行时检查：

- DAG 中无循环。
- 无死胡同阶段。
- 无未定义的 `dependsOn` 引用。
- 无未解析的插值占位符（开启 `strictInterpolation` 时）。
- 预算不会溢出。

这是优先选择声明式图而非命令式脚本的主要原因：你可以在花钱之前*证明*计划的结构属性。

### 3. 上下文隔离

使用原始子代理时，每个 transcript 都会涌入你的上下文。使用 taskflow 时，每个中间输出都保存在运行时中。只有最终阶段输出才会返回到你的对话。

这意味着你可以并行运行 50 个子代理，而上下文窗口仍然很小。

### 4. 跨会话续跑

每个完成的阶段都会写入磁盘。失败或被中止的运行可以在稍后通过 `/tf resume <runId>` 继续——已缓存阶段自动跳过，只有剩余工作消耗 token。

## 何时使用 taskflow

当以下情况时使用 taskflow：

- 工作有**多个依赖步骤**。
- 需要在多个条目上**fan out**。
- 希望最终输出前有**质量门**。
- 计划应该**按名称复用**。
- 想要**成本控制**或**人工审批**。
- 计划可能由另一个模型**生成**，并且必须在运行前被检查。

对于一次性的单步骤委托，内置子代理工具已经足够完美。

## 整体架构

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   作者定义      │ ──▶ │  运行时验证      │ ──▶ │  运行时执行     │
│  taskflow JSON  │     │ 在 token 前验证  │     │  按阶段执行     │
└─────────────────┘     └──────────────────┘     └─────────────────┘
                               │                          │
                               ▼                          ▼
                        ┌─────────────┐          ┌─────────────┐
                        │ Mermaid SVG │          │  仅最终输出  │
                        │   报告      │          │             │
                        └─────────────┘          └─────────────┘
```
